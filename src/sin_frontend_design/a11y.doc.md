# a11y.py

**What:** WCAG 2.2 AA compliance checker.

**Why it exists:** A11y is non-negotiable. The checker runs on every review
and produces a deterministic score the agent can act on.

**Exports:**
- `A11yChecker` — main class.
- `A11yFinding` — one issue (`rule`, `severity`, `message`, `criterion`, `line_hint`).
- `A11yReport` — full report (`ok`, `score`, `findings`).
- `A11yChecker.check_contrast(fg, bg)` — pure math: returns WCAG ratio and
  pass/fail for AA-normal, AA-large, AAA-normal.

**WCAG criteria covered:**
| Rule | Criterion |
|------|-----------|
| `img-alt`           | 1.1.1 Non-text Content |
| `html-lang`         | 3.1.1 Language of Page |
| `page-title`        | 2.4.2 Page Titled |
| `heading-skip`      | 1.3.1 Info and Relationships |
| `heading-first-not-h1` | 1.3.1 Info and Relationships |
| `link-empty`        | 2.4.4 Link Purpose (In Context) |
| `link-generic`      | 2.4.4 Link Purpose (In Context) |
| `div-onclick`       | 2.1.1 Keyboard |
| `input-label`       | 1.3.1 Info and Relationships |

**Scoring:** `100 − 7*errors − 3*warnings − 1*info`, floored at 0.
A report is `ok` when score ≥ 80 and no errors.

**Files that import this:** `mcp_server.py`, `tests/test_a11y.py`.

**Caveats:**
- The contrast function only accepts `#rrggbb` or `#rgb`. For HSL / named
  colors, convert first.
- The checker is regex-based, not a full HTML parser; for deep AST analysis
  plug in `html5lib` or `lxml` separately.
